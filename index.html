we help in 3 ways:

  2. ai guiderails .. claude will deliver way better outcomes because we provide a bunch of common preset code templates for claude to build the things you need to do most often - plus we give claude specific guidance on what to do (and what not to do)

our security philosophy (important)

andrew trusts anthropic's "public benefit corporation" status, but he understands that most people feel uneasy about ai security in general

the best results for building with claude's assistance come from working in "yolo" mode, where it has the "god" privileges to consider and do more things on our behalf

this comes with the risk that claude could deliberately or inadvertently damage our junk

this seems like a conundrum but actually there's a way to achieve both: we just keep our personal stuff totally separate from the sandbox where we're playing with ai. this is actually pretty simple to achieve if we:

  • create a dedicated gmail account for your initial learning journey with claude. this avoids the gnarly worry when you want to give your app access to a calender or gmail but feel uncomfortable giving carte-blanche to your personal data. this way claude has (and you have) free reign in an empty account
  • create a dedicated mac user (without admin rights) on your local machine, so claude can't see any of your personal data. this way claude and your apps can't install anything globally and can't see outside the sandbox.

you'll need a mac

sorry. if you only have windows at home then you'll need to consider a second hand mac (apple silicon). the good news is you will be fine with low spec. andrew uses a 2023 macbook air M2 with 16gb ram and it's never been slow

before you start

these one-time steps can't be automated

  1. sign up for a new gmail account
    • important .. don't reuse your personal account
    • you'll have more freedom to use email, calendars, drive etc if it's a dedicated, separate account
    • plus - you'll get new ~ AUD $400 free credits for google cloud
      .. not that you'll need them
    • if google refuses you mobile verification with "used too many times" consider a one-time $15 sim. you don't need to use this number post-registration
  2. accept google cloud terms & activate free trial

    if you have opted to re-use an old gmail account, you'll have to muddle through the process of accepting terms and enabling billing in google cloud instead of these steps. don't worry, you're not going to be incurring significant costs

    • use an incognito window with no google accounts signed in
    • go here: console.cloud.google.com
    • set up 2fa if prompted
    • accept terms
    • fill in contact & payment method
      .. credit card isn't auto-charged post trial
    • "start free"
    • "close" on the welcome prompt - and you're done
      .. leave the window open for your next step
  3. accept firebase terms
    • use the same incognito window from above
    • go here: console.firebase.google.com
    • get started / create new
    • accept terms
    • call it anything you want, we'll be deleting it in a sec
    • leave all switches/choices off (developer program, gemini, analytics etc)
    • when that's done delete the project
      settings (top-left) -> general -> delete project (bottom)
  4. on your mac create a new (standard) user called "if"
    • important .. later setup won't work without this
    • system settings -> users & groups -> add user
    • type: standard, account name: if
    • optional: system settings -> menu bar -> fast user switching
    • then switch to the new user
    • optional: fast user switching, so you can swap back again easily
    • resist tweaking/installing stuff until after setup runs
